Verification can't see this

Passes every gate. Still wrong.


public function invoiceTotal(array $lines): float
{
    $total = 0.0;
    foreach ($lines as $line) {
        $price = $line['unit_price_cents'] / 100;            // money → float
        $total += round($price * $line['qty'] * 1.0825, 2);  // 8.25% tax, per line
    }
    return round($total, 2);
}
// phpstan: green · phpunit: green · lint: green · spec: met
// 3 lines: $64.92 ✓   ·   10k lines: off by $8.25   ·   100k: off by $82.50

↩ no machine flagged it, only validation against the contract does

This code passes every gate — phpstan, phpunit, lint, the spec. And it's still wrong. It holds money in floats and rounds tax on every line, so the rounding error compounds: nothing at three lines, eight dollars off at ten thousand, eighty-two fifty at a hundred thousand. No machine flagged it. Only validation against the contract catches it. — backstage — THE DEMO (~2 min). The instrument: the harness

Evidence over trust

Tamper-proof tests signed

Visual proof Playwright

Anatomy of a Bet

Three parts, always

H

Hypothesis

What we believe will move a signal: "loyalty points lift repeat purchases."

S

Signal validates

The measurable proof, and the validation target: "repeat-purchase rate, not points issued."

G

Guardrail

The risk envelope: "no retroactive backfill; liability must reconcile with revenue."

Every Bet has three parts. A hypothesis — what we believe will move a signal. The signal itself — the measurable proof, and the validation target. And a guardrail — the risk we won't cross. — backstage — Pair each Bet with a one-screen ADR (Title / Status / Context / Decision / Consequences); see scd-demo doc/adr/0001.
